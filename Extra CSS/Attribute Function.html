Understanding HTML Attributes

Global Attributes

These attributes can be used on any HTML element. Examples include id, class, style, and data-*.

This is an example of global attributes.

Explanation:

  • The id attribute assigns a unique identifier to the element.
  • The class attribute assigns the element to a class, enabling CSS styling.
  • The style attribute applies inline CSS.
  • The data-* attribute stores custom data.

Event Attributes

These attributes trigger JavaScript functions based on user interactions, such as onclick, onmouseover, and onchange.

Explanation:

  • The onclick attribute triggers an alert when the button is clicked.

Form Attributes

These attributes define form behavior, such as action, method, required, and placeholder.

Explanation:

  • The action attribute specifies the form submission URL.
  • The method attribute specifies the HTTP method (GET or POST).
  • The required attribute makes the field mandatory.
  • The placeholder attribute provides a hint to the user.

ARIA (Accessible Rich Internet Applications) Attributes

These attributes improve accessibility for users with disabilities. Examples include aria-label, aria-hidden, and role.

Explanation:

  • The role attribute defines the element's role (e.g., button).
  • The aria-label provides a label for assistive technologies.
  • The tabindex attribute makes the element focusable.

Bootstrap-Specific Attributes

These attributes add predefined Bootstrap styling and functionality, such as data-bs-toggle for modals or collapses.

Explanation:

  • The data-bs-toggle attribute activates Bootstrap's JavaScript components (e.g., collapse).
  • The data-bs-target attribute specifies the target element.
